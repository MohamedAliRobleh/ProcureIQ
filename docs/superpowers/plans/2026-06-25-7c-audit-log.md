# Lot C — Admin Audit Log Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Record sensitive org actions (clear, reset, seed, export) in an append-only `AuditLog` and show the recent entries in the admin page.

**Architecture:** A new `AuditLog` model + a `buildAuditData` helper. The three transactional org endpoints append one `auditLog.create` to their existing `$transaction` (atomic with the action); `export` logs best-effort after its read. A new admin-only `GET /api/org/audit` lists the latest 50, surfaced in an "Activity log" card on the Admin page.

**Tech Stack:** Prisma + Neon, Vercel functions, Clerk auth (`requireOrgAdmin`/`requireAuth`), React, Vitest + Testing Library.

## Global Constraints

- `action` ∈ `'org.clear' | 'org.reset' | 'org.seed' | 'org.export'`; `actorId` = `req.auth.userId`.
- The audit `create` is the LAST element of each `$transaction([...])` array (clear/reset/seed), so it commits iff the action commits. `export` logs best-effort: `await prisma.auditLog.create(...).catch(() => {})` — a log failure must not break the export.
- `GET /api/org/audit` is `requireOrgAdmin`, org-scoped, `orderBy createdAt desc`, `take: 50`; non-GET → `405`.
- `buildAuditData({ orgId, actorId, action })` returns `{ id, orgId, actorId, action }` with `id` prefixed `audit_`.
- Admin "Activity log" fetch is guarded on `isAdmin` inside the effect; the component stores the response only when `Array.isArray` (state initialised `[]`), so existing fetch-mocked Admin tests stay green.
- Do NOT change the existing clear/reset/seed/export success semantics other than appending the audit entry.
- Tests must stay green (currently 379). api half: `npx vitest run api/`; src half serial: `npx vitest run src/ --no-file-parallelism`. Match existing style (ESM, no semicolons, 2-space indent).
- Commit after each task with a `feat(7c-audit):` / `test(7c-audit):` prefix.

// Builds the `data` object for an audit-log row. Returned (not awaited) so it can be

- [ ] **Step 3: Write the implementation**

Create `api/org/audit.js`:

```js
import { prisma } from '../_lib/prisma.js'
import { requireOrgAdmin } from '../_lib/auth.js'

// Admin-only: the org's recent audit log, newest first.
async function handler(req, res) {
  if (req.method !== 'GET') {
    res.setHeader('Allow', 'GET')
    return res.status(405).json({ error: 'Method not allowed' })
  }
  try {
    const entries = await prisma.auditLog.findMany({
      where: { orgId: req.auth.orgId },
      orderBy: { createdAt: 'desc' },
      take: 50,
    })
    return res.status(200).json(entries)
  } catch (e) {
    return res.status(500).json({ error: e.message })
  }
}

export default requireOrgAdmin(handler)
```
